# 모임 매니저 MVP PRD

## 핵심 정보

**목적**: 소규모 모임(5~20명) 주최자가 이벤트 생성, 참여자 관리, 공지 발송을 한 곳에서 처리할 수 있도록 운영 부담을 줄인다.
**사용자**: 수영, 헬스, 친구 모임 등을 직접 주최하거나 참여하는 일반 사용자

---

## 사용자 여정

```
1. 초대 링크 수신 (비로그인 상태)
   ↓ 초대 코드 URL 접속

2. 초대 코드 미리보기 페이지
   ↓ "참여 신청" 버튼 클릭

   [비로그인] → 로그인 페이지 → 로그인 완료 → 초대 코드 미리보기 페이지로 복귀
   [로그인]   → 참여 신청 처리

   [자유참여 이벤트]  → 즉시 승인 → 이벤트 상세 페이지
   [승인형 이벤트]    → 대기(pending) 상태 → 이벤트 상세 페이지 (제한된 뷰)

3. 이벤트 상세 페이지
   ↓ 승인된 참여자만 공지 열람 탭 활성화

   [주최자]           → 이벤트 수정 / 참여자 관리 / 공지 작성 접근 가능
   [승인된 참여자]    → 공지 목록 열람 / 댓글 작성 가능
   [대기 참여자]      → 기본 이벤트 정보만 열람

4. 주최자 - 이벤트 생성 플로우
   내 이벤트 목록 페이지 → "새 이벤트 만들기" 클릭
   → 이벤트 생성 페이지 → 정보 입력 후 저장
   → 이벤트 상세 페이지 (초대 코드 확인 및 공유)

5. 주최자 - 참여자 관리 플로우
   이벤트 상세 페이지 → "참여자 관리" 클릭
   → 참여자 관리 페이지 → pending 목록에서 승인/거절
   → 처리 완료 후 동일 페이지 유지

6. 공지 작성 플로우 (주최자)
   이벤트 상세 페이지 → 공지 탭 → "공지 작성" 클릭
   → 공지 작성 페이지 → 저장
   → 공지 목록 페이지로 이동

7. 공지 열람 및 댓글 (승인된 참여자)
   이벤트 상세 페이지 → 공지 탭 → 공지 항목 클릭
   → 공지 상세 페이지 → 댓글 작성 또는 열람
```

## 기능 명세

### 1. MVP 핵심 기능

| ID       | 기능명              | 설명                                                                                                             | MVP 필수 이유                   | 관련 페이지                                          |
| -------- | ------------------- | ---------------------------------------------------------------------------------------------------------------- | ------------------------------- | ---------------------------------------------------- |
| **F001** | 이벤트 생성         | 제목, 설명, 날짜/시간, 장소, 최대 인원, 참여 방식(open/approval) 입력 후 이벤트 생성. 초대 코드(8자리) 자동 생성 | 서비스의 핵심 데이터 단위 생성  | 이벤트 생성 페이지                                   |
| **F002** | 이벤트 수정/취소    | 주최자가 이벤트 정보를 수정하거나 상태를 cancelled로 변경                                                        | 이벤트 정보 변경 시 필수        | 이벤트 수정 페이지, 이벤트 상세 페이지               |
| **F003** | 내 이벤트 목록      | 내가 주최한 이벤트와 참여한 이벤트를 탭으로 구분하여 표시                                                        | 이벤트 진입점, 전체 서비스 허브 | 내 이벤트 목록 페이지                                |
| **F004** | 초대 코드 미리보기  | 비로그인 포함 누구나 초대 코드로 이벤트 기본 정보 열람. 참여 신청 버튼 제공                                      | 참여자 유입의 유일한 경로       | 초대 코드 미리보기 페이지                            |
| **F005** | 참여 신청/취소      | 로그인한 사용자가 이벤트에 참여 신청. 자유참여는 즉시 승인, 승인형은 pending 상태로 대기. 참여 취소 가능         | 참여 시스템의 핵심              | 초대 코드 미리보기 페이지, 이벤트 상세 페이지        |
| **F006** | 참여자 승인/거절    | 주최자가 pending 상태 신청자를 개별 승인 또는 거절. 승인/최대 인원 현황 표시                                     | 승인형 이벤트 운영 필수         | 참여자 관리 페이지                                   |
| **F007** | 공지 작성/수정/삭제 | 주최자가 이벤트 공지를 작성, 수정, 삭제                                                                          | 이벤트 내 소통 핵심 기능        | 공지 작성 페이지, 공지 수정 페이지, 공지 상세 페이지 |
| **F008** | 공지 열람           | 승인된 참여자와 주최자만 공지 목록 및 상세 내용 열람                                                             | 승인된 멤버 전용 소통 공간      | 공지 목록 페이지, 공지 상세 페이지                   |
| **F009** | 댓글 작성/삭제      | 승인된 참여자와 주최자가 공지에 댓글 작성. 본인 댓글 삭제 가능                                                   | 참여자 간 소통                  | 공지 상세 페이지                                     |

### 2. MVP 필수 지원 기능

| ID       | 기능명           | 설명                                                                                                                   | MVP 필수 이유                        | 관련 페이지                    |
| -------- | ---------------- | ---------------------------------------------------------------------------------------------------------------------- | ------------------------------------ | ------------------------------ |
| **F010** | 기본 인증        | 회원가입/로그인/로그아웃 (이메일+비밀번호, Google OAuth)                                                               | 이벤트 참여 및 주최를 위한 최소 인증 | 로그인 페이지, 회원가입 페이지 |
| **F011** | 이벤트 상세 조회 | 로그인한 사용자가 자신이 관련된 이벤트 상세 정보 열람. 역할(주최자/승인참여자/대기)에 따라 탭 및 액션 버튼 조건부 표시 | 이벤트 내 모든 기능의 진입점         | 이벤트 상세 페이지             |

- 정산/결제

## 메뉴 구조

```
[비로그인 사용자]
├── 초대 코드 미리보기 (URL 직접 접근)
│   └── 기능: F004 (이벤트 기본 정보 열람, 로그인 유도)
└── 인증
    ├── 로그인   - F010
    └── 회원가입 - F010

[로그인 사용자 - 공통 헤더]
├── 내 이벤트 목록 (홈)
│   └── 기능: F003 (주최한 이벤트 / 참여한 이벤트 탭)
├── 새 이벤트 만들기
│   └── 기능: F001 (이벤트 생성)
└── 로그아웃

[이벤트 상세 - 주최자 메뉴]
├── 이벤트 정보        - F011 (기본 정보 열람)
├── 참여자 관리        - F006 (승인/거절)
├── 공지              - F008 (공지 목록 열람)
│   └── 공지 작성      - F007
└── 이벤트 수정/취소   - F002

[이벤트 상세 - 승인된 참여자 메뉴]
├── 이벤트 정보        - F011 (기본 정보 열람)
├── 공지              - F008 (공지 목록 열람)
└── 참여 취소          - F005
```

## 페이지별 상세 기능

### 로그인 페이지 / 회원가입 페이지

> **구현 기능:** `F010` | **인증:** 비로그인 전용 (로그인 시 내 이벤트 목록 페이지로 리디렉션)

| 항목            | 내용                                                                                                                                  |
| --------------- | ------------------------------------------------------------------------------------------------------------------------------------- |
| **역할**        | 서비스 이용을 위한 최소 인증 처리 전용 페이지                                                                                         |
| **진입 경로**   | 헤더 로그인/회원가입 클릭, 초대 코드 미리보기 페이지에서 "참여 신청" 클릭 시 리디렉션                                                 |
| **사용자 행동** | 이메일+비밀번호 또는 Google OAuth로 로그인 또는 회원가입                                                                              |
| **주요 기능**   | - 이메일/비밀번호 로그인 및 회원가입<br>- Google OAuth 로그인/회원가입<br>- 비밀번호 재설정 링크 제공<br>- **로그인 / 회원가입** 버튼 |
| **다음 이동**   | 성공 → 내 이벤트 목록 페이지 (초대 코드로 진입한 경우 해당 미리보기 페이지로 복귀)                                                    |

| --------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **역할**        | 로그인 사용자의 서비스 허브. 주최/참여 이벤트 전체 파악                                                                                                                                                                                                                 |

| **다음 이동**   | 이벤트 카드 클릭 → 이벤트 상세 페이지, 새 이벤트 만들기 → 이벤트 생성 페이지                                                                                                                                                                                            |

| **진입 경로**   | 내 이벤트 목록 페이지에서 "새 이벤트 만들기" 버튼 클릭                                                                                                                                                                                          |

### 이벤트 상세 페이지

> **구현 기능:** `F002`, `F005`, `F011` | **인증:** 로그인 필수

| 항목            | 내용                                                                                                                                                                                                                                                                                                                                                                                           |
| --------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **역할**        | 이벤트의 중심 허브. 역할에 따라 조건부로 액션 버튼과 탭을 노출하는 페이지                                                                                                                                                                                                                                                                                                                      |
| **진입 경로**   | 내 이벤트 목록에서 카드 클릭, 초대 코드 미리보기에서 참여 신청 완료 후 리디렉션                                                                                                                                                                                                                                                                                                                |
| **사용자 행동** | 이벤트 기본 정보 열람, 역할에 따라 수정/취소/참여 취소/참여자 관리/공지 탭 이동                                                                                                                                                                                                                                                                                                                |
| **주요 기능**   | - 이벤트 기본 정보 표시: 제목, 설명, 날짜, 장소, 승인 인원 / 최대 인원, 참여 방식<br>- [주최자 전용] 초대 코드 복사 버튼, **이벤트 수정** 버튼, **이벤트 취소** 버튼<br>- [주최자 전용] **참여자 관리** 버튼 (참여자 관리 페이지로 이동)<br>- [주최자 + 승인된 참여자] **공지** 탭 (공지 목록 페이지로 이동)<br>- [승인된 참여자 전용] **참여 취소** 버튼 (취소 확인 후 cancelled 상태로 변경) |
| **다음 이동**   | 이벤트 수정 → 이벤트 수정 페이지, 참여자 관리 → 참여자 관리 페이지, 공지 탭 → 공지 목록 페이지                                                                                                                                                                                                                                                                                                 |

### 초대 코드 미리보기 페이지

> **구현 기능:** `F004`, `F005` | **인증:** 비로그인 열람 가능, 참여 신청은 로그인 필수

| 항목            | 내용                                                                                                                                                                                                                                                                                                                   |
| --------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **역할**        | 외부에서 초대 링크를 받은 사람이 이벤트를 확인하고 참여 신청하는 공개 페이지                                                                                                                                                                                                                                           |
| **진입 경로**   | `/invite/[code]` URL 직접 접속 (주최자가 공유한 링크)                                                                                                                                                                                                                                                                  |
| **사용자 행동** | 이벤트 기본 정보 확인 후 참여 신청 버튼 클릭. 비로그인이면 로그인 페이지로 이동                                                                                                                                                                                                                                        |
| **주요 기능**   | - 이벤트 기본 정보 표시: 제목, 설명, 날짜, 장소, 참여 방식, 현재 승인 인원 / 최대 인원<br>- 유효하지 않은 코드 접근 시 404 안내<br>- 이미 신청/승인된 상태이면 상태 표시 및 신청 버튼 비활성화<br>- 최대 인원 초과 시 "정원 마감" 표시<br>- **참여 신청** 버튼 (비로그인 → 로그인 페이지 리디렉션, 로그인 → 즉시 처리) |

## 구현 단계

### Phase 1 - DB 스키마 + 타입 정의

- Supabase에서 `events`, `event_participants`, `announcements`, `announcement_comments` 테이블 생성
- 각 테이블에 RLS 정책 설정 (주최자/참여자 권한 분리)
- `invite_code` 중복 방지 unique 인덱스
- TypeScript 타입 생성 (`npx supabase gen types typescript`)

**RLS 핵심 규칙 요약**

| 테이블                | SELECT                              | INSERT                              | UPDATE                     | DELETE           |
| --------------------- | ----------------------------------- | ----------------------------------- | -------------------------- | ---------------- |
| events                | 누구나 (invite_code 조회 포함)      | 로그인 사용자                       | host_id = 본인             | host_id = 본인   |
| event_participants    | 본인 행 또는 host_id = 본인         | 로그인 사용자 (본인 행)             | host_id = 본인 (승인/거절) | 본인 행 (취소)   |
| announcements         | host_id = 본인 또는 approved 참여자 | host_id = 본인                      | author_id = 본인           | author_id = 본인 |
| announcement_comments | host_id = 본인 또는 approved 참여자 | host_id = 본인 또는 approved 참여자 | author_id = 본인           | author_id = 본인 |

### Phase 2 - 이벤트 CRUD

- 이벤트 생성 페이지: 폼 + 서버 액션 + invite_code 생성 로직
- 내 이벤트 목록 페이지: 주최/참여 탭 구분
- 이벤트 상세 페이지: 역할 기반 조건부 UI
- 이벤트 수정 페이지: prefill + 서버 액션

### Phase 3 - 초대 코드 + 참여 시스템

- 초대 코드 미리보기 페이지 (`/invite/[code]`): 공개 접근, 로그인 유도
- 참여 신청 서버 액션: join_policy에 따라 즉시 승인 또는 pending 처리
- 참여 취소 서버 액션

### Phase 4 - 공지 + 댓글

- 공지 목록 페이지: 승인 여부 체크 후 접근 제어
- 공지 작성/수정 페이지 + 서버 액션
- 공지 상세 페이지 + 댓글 목록/작성/삭제 서버 액션

### Phase 5 - 참여자 관리 UI

- 참여자 관리 페이지: 상태별 탭 + 승인/거절 서버 액션
- 현황 (승인 인원 / 최대 인원) 표시
